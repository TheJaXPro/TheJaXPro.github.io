可解释性

  • 有些问题并不需要解释
  • 可解释性的根源来源于：问题形式的不完整性

可解释性的分类：

  • 1.本质的：限制机器学习模型的复杂性
  • 2.事后的：模型训练结束之后解释
  • 1.特定于模型：分析模型内部
  • 2.模型无关：通常通过分析特征输入输出来分析
  • 1.局部：哪个特征最重要-》“为什么这是一只猫”
  • 2.全局：整个模型的行为-》“你认为猫是怎么样的”

一.Local Explanation:Explain the Decision

1.Saliency Map

对于一个模型，直接提取对应label对于输入的梯度即可。梯度越大对应的像素点越重要，直接将梯度可视化就可以得到解释的内容。

2.Integrated gradient（gradient-based mothod）

直接获得对于输入的梯度，其实对于解释性会有很大的限制——因为存在梯度饱和问题。作者提出了一种叫做“梯度积分”的方法，就是使用如下公式：

3.DeepLIFT（Backpropagation-based mothod）

Backpropagation-based mothod比gradient-based mothod好的一点在于，它本身并没有采用model反向传播的那种链式法则，而是自己定义了一些因子，通过定义的计算方法，求得最终反向传播的结果。求的不再是梯度，而是gradient*input，即实际效应。

三.USING A MODEL TO EXPLAIN ANOTHER

1.Local Interpretable Model-Agnostic Explanations (LIME)

使用线性模型在给定数据点的左右去做一个线性拟合，对于图片可以将图片切成不同的模块，分别放入网络之中，再使用辅助变量表示某个模块是否存在。

  1. Given a data point you want to explain
  2. Sample at the nearby
  3. Fit with linear model (or other interpretable models)
  4. Interpret the linear model

把BlackBox的输出作为决策树的软标签来训练一个决策树，但是如果决策树太大，那么结果也是很难理解的。由于决策树的复杂度本身是不可导的，无法作为一个正则项加入到损失函数之中。这里作者使用了一个巧妙的方法：

使用一个神经网络来得到决策树和其复杂度之间的映射关系，锁定这个神经网络，就可以得到一个可以求导的正则项。将这个正则项加入到损失函数之中，就可以取得可解释性的决策树。
